kanban/ に新しい調査タスクファイルを作成します。

## 引数

$ARGUMENTS

引数としてタスクのファイル名タイトル（英語・ハイフン区切り）を指定できます（例: `/add-kanban search-api-limits`）。
引数がない場合は「知りたいこと」から自動生成します。

## 手順

### 1. コンテンツの収集

ユーザーが同一メッセージ内にすでに「知りたいこと」や「目的」を記載している場合はそれを使用する。

記載がない場合は以下を確認する:

- **知りたいこと**（必須）: 具体的に何を調べたいか
- **目的**（任意）: なぜこの調査が必要か。省略可能である旨を伝える

### 2. タイトルの確定

タイトルは **ファイル名用（英語）** と **見出し用（日本語）** の2種類を用意する。

- `$ARGUMENTS` が指定されている場合: それをファイル名タイトルとして使用する。日本語見出しは「知りたいこと」の内容を要約して生成する
- `$ARGUMENTS` がない場合: 「知りたいこと」の内容から Claude が適切なファイル名（英語・ハイフン区切り）と日本語見出しを生成する

例:
- 知りたいこと:「Enterprise Search の検索 API のレート制限」→ ファイル名 `search-api-rate-limit`、見出し `Enterprise Search 検索 API レート制限`

### 3. 連番の計算

`kanban/[0-9][0-9][0-9][0-9]_*.md` のファイル一覧を Glob で取得し、ファイル名先頭の数字部分から最大値を求め、+1 した値を4桁ゼロパディングする。

- ファイルがひとつもない場合は `0000` から開始
- 例: `0003_connection_report_auth.md` が最大なら次は `0004`

### 4. ファイルの生成

`kanban/{padded}_{ファイル名タイトル（英語）}.md` を Write ツールで作成する。

ファイルの内容:

```
# {日本語タイトル}
## 知りたいこと

{知りたいこと の内容}

## 目的

{目的 の内容（省略時は空行のまま）}
```

### 5. kanban タスクの実行

作成したファイルのパスをユーザーに伝えた後、**そのまま続けて `/kanban` スキルを実行する**。
Skill ツールで `skill: "kanban"`, `args: "{xxxx}"` を呼び出すことで、作成したタスクの調査をただちに開始する。
