## 調査結果

### 1. Org 全体でエンタープライズ検索を有効化する手順

Org オーナーまたは Org 管理者が行う操作。

**手順：**
1. デスクトップのサイドバーにあるオーガナイゼーション名をクリック
2. 「ツールと設定」にマウスポインターを重ねて「オーガナイゼーションの設定」を選択
3. 左側サイドバーの「設定」から「エンタープライズ検索」を選択
4. 「編集」をクリック
5. 「オーガナイゼーション全体でエンタープライズ検索を有効にする」のボックスをチェックして「保存」をクリック

**前提条件・注意事項：**
- OrG オーナーまたは OrG 管理者権限が必要
- Enterprise+ または Enterprise Grid プランの契約が必須
- ほとんどの OrG ではデフォルトで無効
- ただし Google ドライブや OneDrive アプリを使用している場合は自動的に有効になる可能性あり

**無効化する場合：** 同じ手順でボックスをチェック解除して保存

---

### 2. データソースを追加する手順（Org 管理者の作業）

**手順：**
1. オーガナイゼーション名をクリック
2. 「ツールと設定」→「オーガナイゼーションの設定」を選択
3. 「設定」→「エンタープライズ検索」を選択
4. 追加するデータソースの横の「設定する」をクリック
5. 「使用できる人」の横の「編集」をクリック
6. ドロップダウンメニューからオプションを選択
7. 特定の IDP グループまたはメンバーにアクセス権を付与/制限する場合、該当するボックスにチェック
8. 必要に応じて「これらのメンバーが AI 検索時にこのリソースにアクセスできるようにする」のチェックを外す
9. 「保存」をクリック

**サポートされているデータソース（12種類）：**
- Asana
- Box
- Confluence Cloud
- Dropbox
- GitHub
- Google ドライブ
- Google メール
- Jira Cloud
- Microsoft Outlook
- Microsoft Teams
- OneDrive / SharePoint
- Salesforce

**データソースごとの追加設定：**
- **Asana**: Asana オーガナイゼーションの名前を入力して保存
- **Box**: Box Enterprise 管理者設定で「Box Connector for Slack Workflows」を検索し、「使用可能: すべてのユーザー」を選択
- **Confluence Cloud**: Confluence Cloud のサブドメインを入力して保存
- **GitHub**: GitHub オーガナイゼーション名を入力（最大5つ）し、GitHub アプリをインストール
- **Google メール / Microsoft Outlook**: HTML レンダリング用の許可リストでドメインを追加（オプション）
- **Jira Cloud**: Jira サブドメインを入力して保存

### 3. エンドユーザーレベルでの利用方法

ページには「各自のアカウントを連携して、検索結果でそのソースのコンテンツを表示できます」という記述のみ。

**エンドユーザーの操作（個人アカウントの連携）：**
- Org 管理者がデータソースを追加後、メンバーが個人アカウントを連携することで検索結果にそのソースのコンテンツが表示される
- 具体的な連携手順についての記載はこのページにはなく、別記事参照が示されている
- 検索結果には「検索するユーザーがアクセス権を持っているコンテンツのみが含まれる」

---

### 4. データソースを削除する手順

1. オーガナイゼーション名をクリック
2. 「ツールと設定」→「オーガナイゼーションの設定」を選択
3. 「設定」→「エンタープライズ検索」を選択
4. 削除するデータソースを選択
5. 「エンタープライズ検索と AI 回答」横の「編集」をクリック
6. 「従来の検索結果に含める」および「AI 回答の検索でソースとして使用する」のボックスをチェック解除して保存
7. 「無効にする」を選択して確定

### 6. 有効化の全体フロー（統合サマリー）

前回調査（0022, 0024）と今回取得情報を統合すると、Enterprise Search の有効化には以下のステップが必要：

**ステップ 1: 開発者による App の設定**（開発者の作業）
- App マニフェストに `features.search` オブジェクトを追加
- `org_deploy_enabled: true` を設定
- `function_executed` イベントを購読
- 検索結果を返すカスタムステップ関数を実装

**ステップ 2: Org 全体でエンタープライズ検索を有効化**（Org 管理者の作業）
- 管理画面の「設定」→「エンタープライズ検索」から「オーガナイゼーション全体でエンタープライズ検索を有効にする」をチェック
- Enterprise+ / Enterprise Grid プランが必要

**ステップ 3: App のオーグレベルインストール**（Org 管理者の作業）
- api.slack.com/apps の UI、OAuth フロー、または Slack CLI でインストール
- この時点ではまだどのワークスペースでも使えない

**ステップ 4: ワークスペースへの App 追加**（Org 管理者の作業）
- 管理ダッシュボード（`app.slack.com/manage/<enterprise-id>`）→ Integrations → Installed apps から対象ワークスペースへ追加

**ステップ 5: データソースの設定とエンドユーザーへの有効化**（Org 管理者の作業）
- 「エンタープライズ検索」ページからデータソースの「設定する」をクリック
- 「使用できる人」で IDP グループまたは特定メンバーを指定
- AI 検索時のアクセス許可を設定
- → これが「Org Admin によるエンドユーザーへの有効化」に相当する可能性が高い

**ステップ 6: エンドユーザーによる個人アカウントの連携**（エンドユーザーの作業）
- Org 管理者の設定後、エンドユーザーが個人アカウントを連携
- 連携後は検索結果に外部ソースのコンテンツが表示される
- エンドユーザーは個別に無効化も可能（サイドバーで右クリック → Disable、または Manage から）
